Why do I need it?

jQuery was built to abstract away browser idiosyncracies. It is meant for dealing with the DOM, finding elements and changing the UI

AngularJS was built as a layer on top of jQuery, to add MVC concepts to front end engineering. AngularJS gives you data-binding, templating, custom components.

Additionally, AngularJS gives you structures and concepts that apply across various projects, like Controllers, Services and Directives.

Facts

100% Client Side JavaScript

It is not a library like jQuery

It is a full featured SPA Framework

Will not make your application look prettier like Bootstrap or Foundation

SPA (Single Page Apps) - extremely good for very responsive sites

Is it popular?

What does an Angular app look like?

SPAs are applications that provide a fluid UX by loading all necessary code with a single page load in a browser

All the application content is dynamically added to and removed from that one page

One Way Data Binding

This is the data binding used by most classical templating engines, where the developer has to make sure the view is kept in sync with the model and vice-versa.

Two Way Data Binding

Eliminates the need of event binding and handling

Template based on HTML + custom markup is compiled on the browser, producing a live view

Any changes to the view are immediately reflected in the model, and changes to the model are propagated to the view
Other template systems:
  • jade template
  • swig template
  • handlebars template
  • mustache template

Directives

A directive is something that introduces new syntax

Angular Directives are markers on a DOM element which tell the angular compiler to attach specific behavior to it

They are the most important components of any AngularJS application. AngularJS ships with a wide range of directives, but often we need to create application specific ones

 
                    
<html>
    <head></head>
    <body ng-controller="Hello">
        ...
        </body>
</html>
                    
                
                    
function Hello() {
    alert('Hello, I am an Angular controller!');
}

Custom Directives

Angular gives you the power to extend the existing HTML vocabulary by creating new directives

                    
<html>

    </body>

Modules

Angular apps don't have a main method. Instead modules declaratively specify how an application should be bootstrapped

Inside of a module you can create controllers, services, filters and directives

A module can depend on other modules.

Application Bootstrapping

Modules can be created or accessed using the module property of the global angular object

                    
// Create a new angular module myApp that depends on module1 and module2
var app = angular.module('myApp', ['module1', 'module2']);

// Reference an existing module in your code
var module1 = angular.module('module1');

The ng-app directive can be used to bootstrap a module by its name:

                    
<!-- reference the new module -->
<html ng-app="myApp"></html>
It's a very commum mistake thinking that both methods do the same thing.. They do not, one creates a module, the other is a getter for the module.
Modules are to Angular as packages are to Java.
Directives can be of elements or of attributes.

Angular MVC

View Templates

In Angular, templates are written with HTML that contains Angular-specific elements and attributes

Angular combines the template with information from the model and controller to render the dynamic view that the user sees in the browser

The types of Angular elements and attributes you can use in templates are:

  • Directives
  • Expressions
  • Filters
  • Form Controls

Expressions

JavasScript like code snippets that are usually placed in bindings such as {{ expression }} 

                    
<!-- angular expression -->
<p>{{ "I have "}} {{ 4 + 6 }} Euros<p>

<!-- angular expression after evaluation -->
<p>I have 10 Euros<p>

Filters

Change the way expressions are displayed

                    
<!-- usage of the angular uppercase filter -->
<h3>{{ user.name | uppercase }}<h3>

Angular comes with a collection of built-in filters, but it is easy to define custom ones

Works as the shell "|" , you can chain filters.

Controllers

Contains presentation logic and bind the view with the model using scopes

Instantiated only when needed and discarded every time we switch a route or reload a view 
                    
// obtain the application module
var app = angular.module('myApp');

// declare a new controller on this module
app.controller('MyController', function () {
    console.log('I am a useless controller...');
});
By convention the controllers names in Angular start with capital letter.
John Papa (Angular style guide) https://github.com/johnpapa/angular-styleguide/tree/master/a1

Scope

Controllers become useful when they add behaviour and data to the $scope object

In the scope we store everything we need to display or call from the view 
                    
function ChatController($scope) {

    // initialize the chat history with a couple of messages
    $scope.messages = ["I <3 sidra", "Save the whales!"];

    // add a new message to the chat history
    $scope.add = function() {
        $scope.messages.push($scope.newMessage);
        $scope.newMessage = "";
    }
}

Services

Provide a method to keep data around for the lifetime of the app and communicate across controllers

Services are singleton objects that are instantiated only once per app and lazy-loaded

Angular comes with several built-in services, but for any decent complex application it's required to make custom services

$injector

Service responsible for handling all instantiations of Angular components and dependencies, including modules, directives, controllers, etc.

                    
// create a new angular module
var app = angular.module('myApp', []);

// Angular does this for us, but just for fun, let's
// create a new injector for our module, it dependes on angular core
var $injector = angular.injector(['ng', 'myApp']);

// Ask the injector for an instance of the angular log service
var $log = $injector.get('$log');

// Log some message...
$log.info('.. some informative log message ..');
